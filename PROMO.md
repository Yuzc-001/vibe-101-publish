# vibe-101-publish 宣传文案

> 把 Markdown 编辑器，做成真正可发布的内容产品。

---

## 一句话定位

`vibe-101-publish` 是面向公众号创作者的排版与发布工作台。  
它不是“只在编辑区好看”的工具，而是追求“发布后也稳定”的完整链路产品。

在线体验：<https://yuzc-001.github.io/vibe-101-publish/>

## 我们在解决的核心问题

很多工具卡在发布最后一公里：

- 粘贴到公众号后样式丢失、字体错乱。
- 图片、表格、图文混排结构变形。
- LaTeX 公式在目标平台兼容性差。
- 编辑区效果和真实发布效果不一致。

`vibe-101-publish` 的策略是：把“发布稳定性”放在第一优先级，而不是把它当附加功能。

## 核心价值

| 维度 | 价值 |
| --- | --- |
| 发布一致性 | 编辑效果尽量贴近最终发布结果 |
| 微信兼容性 | 针对公众号复制链路持续优化 |
| 表达质量 | 主题、排版节奏、代码、公式可控 |
| 工程可持续 | 功能可验证、可复用、可持续迭代 |

## 能力亮点

### 1. 公众号发布链路优化

- 复制到公众号前进行结构修正与样式补偿。
- 外链图片处理，降低粘贴失真风险。
- 公式链路优化，兼顾渲染质量与平台兼容。

### 2. 创作与排版效率

- 富文本智能粘贴为干净 Markdown。
- 主题系统支持快速切换与个性化微调。
- 支持手机 / 平板 / 桌面多端预览。
- 滚动同步与文档信息面板提升编辑效率。

### 3. 多格式交付能力

- 一键导出 HTML、PDF、DOC、DOCX。
- 适配公众号、知识库、博客、办公文档等发布场景。

## 适用人群

- 技术写作者与长文作者。
- 公众号内容团队和运营团队。
- 高频输出的知识型创作者。
- 对发布品质有明确要求的产品/品牌团队。

## 为什么它已经是“产品”而不是“Demo”

- 有明确场景：内容发布与分发。
- 有明确指标：稳定性、可读性、交付效率。
- 有明确路径：输入 -> 排版 -> 预览 -> 复制/导出 -> 发布。
- 有持续演进：围绕用户痛点进行工程化迭代。
