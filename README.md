# vibe-101-publish - 设计驱动的内容发布引擎

面向公众号、知识库、博客与办公文档的 Markdown 排版与发布系统。  
这不是“模板拼装器”，而是强调表达控制与发布一致性的创作工具。

> 启动方式（Windows）：运行 `start.bat`  
> 启动地址：`http://127.0.0.1:5173/`

## 我们的理念

- 发布一致性优先：编辑体验服务于最终发布效果，而不是只追求编辑器内好看。
- 表达质量优先：主题、字体、间距、代码块、公式都要具备可读性和可控性。
- 微信兼容优先：公众号是核心发布场景，复制链路必须稳定。
- 工程可持续优先：所有能力可复用、可验证、可迭代，而不是一次性脚本堆砌。

## 与上游项目的关系

本项目是围绕“内容排版与发布工作流”的独立新创作与工程重构。  
在演进过程中参考并继承了原项目 [raphael-publish](https://github.com/liuxiaopai-ai/raphael-publish) 的开源基础，在此向原作者及原仓库贡献者致谢。

## 我们的核心差异

- 产品定位从“编辑器功能集合”转向“发布质量引擎”。
- 主题体系重构为必选主题 + 推荐主题 + 自定义主题。
- 导出链路重构为 HTML / PDF / DOC / DOCX + 公众号复制。
- 同步滚动、移动端预览与复制流程按稳定性优先重写。
- 公式链路加入 KaTeX 渲染与公众号兼容策略（HTML 优先，图片兜底）。

## 核心能力

### 1. 魔法粘贴

从飞书、Notion、Word、网页复制富文本后自动净化为 Markdown。  
支持直接粘贴截图或图片（Ctrl/Cmd + V）。

### 2. 多主题排版

提供 30 套风格化主题，覆盖经典、现代、实验视觉风格。  
支持自定义主题主色与风格模式。

### 3. 公众号发布链路

复制到公众号时进行专门兼容处理：

- 外链图片自动转 Base64，避免第三方图片拦截。
- 列表、表格、图文网格做结构修正，降低后台粘贴后错位。
- LaTeX 公式优先走内联样式 HTML，失败时自动回退到图片策略。

### 4. 多端预览与稳定同步

支持手机、平板、桌面预览。  
同步滚动逻辑针对惯性滚动与内容高度变化做了稳定性增强。

### 5. 多格式导出

支持导出 PDF、HTML、DOC、DOCX，并支持复制 Word 富文本。

## 近期更新（2026-03-04）

- 重构同步滚动逻辑，修复手机/平板预览下的错位与漂移问题。
- 新增 KaTeX 公式渲染，支持 `$...$` 与 `$$...$$`。

## 快速开始（Windows）

```bat
start.bat
```

脚本会自动完成依赖安装（如缺失）并启动本地开发服务。

## 本地开发

```bash
pnpm install
pnpm dev
# 或
npm install
npm run dev
```

## 构建部署

```bash
pnpm build
# 或
npm run build
```

构建产物输出到 `dist/`，可部署到 GitHub Pages 或任意静态托管平台。

## 技术栈

- React 18 + TypeScript
- Vite 5
- Tailwind CSS 3
- markdown-it + highlight.js
- KaTeX
- turndown
- html2pdf.js
