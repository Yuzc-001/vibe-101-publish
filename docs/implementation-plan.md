# 实施进展与下一步

这份文档不再记录“准备做什么”，而是记录当前这轮迭代已经落地了什么、还值得继续打磨什么。  
当前时间点：`2026-03`

---

## 项目状态

`vibe-101-publish` 已从“以公众号为主的排版工具”演进为“面向公众号、知乎、Word 与社交卡片的发布工作台”。

本轮最核心的变化有三条：

1. 知乎复制链路重写
2. Markdown 表格 / 公式 / 代码渲染链修复
3. 卡片模式升级为卡片工作台

---

## 已完成能力

## Phase A：发布链路升级

### 公众号复制

- 保留原有 `wechatCompat` 兼容链路
- 继续作为主发布按钮
- 适合公众号后台直接粘贴发布

### 知乎复制

- 新增独立 `zhihuCompat` 适配器
- 不再复用预览 DOM，而是走 `Markdown -> 知乎专用 HTML`
- 公式转为知乎可识别结构
- 表格单元格中的公式、代码、链接做单独处理
- 浏览器不支持富文本复制时，降级为知乎 Markdown 导入文件

### Word 与通用导出

- 复制到 Word
- 导出 `PDF`
- 导出 `HTML`
- 导出 `Word .docx`
- 导出 `Word .doc`

---

## Phase B：渲染稳定性修复

### 表格链路修复

- 主 Markdown 渲染链支持更稳定的表格解析
- 修复表格单元格中的公式被错误吞掉或转义的问题
- 让预览、复制到知乎、复制到其他富文本目标时的结果更接近

### 公式与代码共存问题修复

- 数学预处理跳过 fenced code
- 避免代码中的符号误伤 Markdown 结构
- 保持行内公式、块级公式、代码块三者的边界更清晰

### 卡片截图前的清洗

- 清理坏图和空容器，减少截图中的大白块
- 针对社交卡片模式收敛图片尺寸
- 超长内容自动缩放，并给出“建议拆卡”提示

---

## Phase C：卡片工作台

### 导出模式

- `公众号长图`
  - 宽度 `750px`
  - 自适应长图
  - 适合教程、复盘、深度长文

- `社交卡片`
  - 尺寸 `440 x 586`
  - 固定比例
  - 更适合小红书、朋友圈、图文封面

### 卡片工作流

- 通过 Markdown 中的 `---` 拆分卡片
- 支持封面卡、内容卡、尾页卡
- 支持页码显示
- 支持当前卡预览、全部卡浏览与主题切换
- 支持下载当前 PNG、下载全部 PNG、ZIP 打包下载

### 社交卡片主题

已新增 6 套社交卡片专属主题：

- 小红书便签
- 弥散渐变
- 细线边框
- 深夜黑板
- 格子纸
- Apple 便签

### 交互层升级

- 卡片模式升级为卡片工作台
- 左侧是导出决策与卡片队列
- 右侧是大预览和主题选择
- 当前卡优先渲染，避免首次打开就卡死
- 预览区尺寸、滚动与布局已做多轮优化

## 下一步最值得继续做的方向

### 1. 智能拆卡

当前卡片拆分主要依赖 `---`。  
下一步可以考虑按标题、段落长度、图片位置自动建议拆卡，让长文转卡片更省心。

### 2. 预览放大与审稿模式

卡片工作台已经有大预览，但仍可继续增强：

- 增加放大查看
- 增加仅预览模式
- 增加“只看当前卡”与“整组浏览”切换

### 3. 平台级验收样例库

可以补一套更正式的回归样例：

- 公式样例
- 表格样例
- 多图样例
- 长内容样例
- 复杂引用与代码块样例

这样后续每次改知乎链路或卡片导出时，验收会更稳。
